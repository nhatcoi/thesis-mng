# 2.2.3 Thiết kế API Chi tiết – ThesisHub

Tài liệu mô tả thiết kế từng API theo định dạng: STT, URL, Phương thức, Mô tả chi tiết, Request/Response, Phân quyền.

**Định dạng Response chung (ApiResponse&lt;T&gt;):**
```json
{
  "success": true,
  "message": "Thông báo (tùy chọn)",
  "data": { /* payload */ },
  "timestamp": "2026-03-14T12:00:00+07:00"
}
```

---

## 1. Xác thực (Auth)

| STT | URL | Phương thức | Mô tả chi tiết |
|-----|-----|-------------|----------------|
| 1 | `/api/auth/check` | GET | **Kiểm tra quyền truy cập.** Kiểm tra user đã đăng nhập, đồng bộ từ OIDC claims, và kiểm tra điều kiện được phép vào hệ thống (allowed). Trả về `AuthCheckResponse`: `allowed`, `reasonCode`, `message`, `localUserId`, `localRole`, `tokenRoles`. 401 nếu chưa đăng nhập; 403 nếu không đủ quyền. **Trạng thái:** Đã code. |
| 2 | `/api/auth/me` | GET | **Lấy thông tin người dùng hiện tại.** Trả về thông tin đầy đủ: `sub`, `preferred_username`, `email`, `name`, `local_user_id`, `local_roles`, `facultyId`, `facultyName`, `majorName`, `managedMajorName`, `majorCode`, `managedMajorCode`, `roles` (từ token). Yêu cầu đăng nhập và qua kiểm tra auth gate. **Trạng thái:** Đã code. |
| 3 | `/api/auth/userinfo` | GET | **Lấy thông tin user từ OIDC.** Trả về: `sub`, `email`, `name`, `roles` (từ token). Dùng cho client OAuth2. **Trạng thái:** Đã code. |

## 2. Người dùng (User)

| STT | URL | Phương thức | Mô tả chi tiết |
|-----|-----|-------------|----------------|
| 4 | `/api/users` | POST | **Tạo người dùng.** Body: `UserCreateRequest` – `username`, `email`, `firstName`, `lastName`, `phone`, `roles`, `majorCode`, `cohort`, `className` (SV), `facultyCode`, `managedMajorCode`, `maxStudentsPerBatch` (GV). Phân quyền: ADMIN, TRAINING_DEPT. **Trạng thái:** Đã code. |
| 5 | `/api/users` | GET | **Danh sách người dùng (phân trang, lọc).** Query: `search`, `role`, `facultyId`, `majorCode`, `page`, `size`, `sort`. Trả về `Page<UserResponse>`. Phân quyền: ADMIN, TRAINING_DEPT. **Trạng thái:** Đã code. |
| 6 | `/api/users/{id}` | PUT | **Cập nhật người dùng.** Body: tương tự `UserCreateRequest` (các trường cập nhật). Phân quyền: ADMIN, TRAINING_DEPT. **Trạng thái:** Chưa code. |
| 7 | `/api/users/{id}` | DELETE | **Xóa người dùng.** Path: `id` (UUID). Phân quyền: ADMIN, TRAINING_DEPT. **Trạng thái:** Chưa code. |
| 8 | `/api/users/{id}` | GET | **Chi tiết người dùng.** Trả về `UserResponse`. Phân quyền: ADMIN, TRAINING_DEPT. **Trạng thái:** Chưa code. |
